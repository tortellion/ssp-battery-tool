## Setup (~15 minutes)

### Step 1 — Supabase (free account)

1. Go to [supabase.com](https://supabase.com) → New project
2. Name it `ssp-battery-tool`, pick a region close to your team
3. Once created: go to **SQL Editor** → **New query** → paste the contents of `supabase/schema.sql` → Run
4. Go to **Settings → API** → copy:
   - Project URL (looks like `https://xxxxx.supabase.co`)
   - `anon` public key

### Step 2 — Vercel (free account)

1. Go to [vercel.com](https://vercel.com) → Add New Project
2. Import from GitHub (push this repo first) or use the Vercel CLI
3. Add these **Environment Variables** in Vercel project settings:
   ```
   NEXT_PUBLIC_SUPABASE_URL     = your Supabase project URL
   NEXT_PUBLIC_SUPABASE_ANON_KEY = your Supabase anon key
   ANTHROPIC_API_KEY             = sk-ant-your-key-here
   ```
4. Deploy. Vercel gives you a URL like `ssp-battery-tool.vercel.app`

### Step 3 — Add your team

1. Open the deployed URL
2. Click **Create account** and sign up with your email
3. Check your email for the confirmation link
4. Share the URL with Tom, Derek, etc. — they create accounts the same way

### Step 4 — Custom domain (optional)

In Vercel project settings → Domains → add `battery.solidstatepower.com` or any subdomain.

## How It Works

| Layer | Technology | What It Does |
|---|---|---|
| Frontend | React + Next.js | All the UI you've been using |
| API routes | Next.js serverless functions | Calls Anthropic — API key never in browser |
| Auth | Supabase Auth | Email/password, session management |
| Database | Supabase (PostgreSQL) | Library, history, settings |
| Hosting | Vercel | Serves the app, auto-deploys on push |

**Shared data:** One library record shared across all users. When an engineer imports the catalog, everyone sees the update within seconds. History is also shared — any team member can see configurations generated by others.

**API key:** Lives in Vercel environment variables. Never exposed to the browser. Never typed by users.

## File Structure

```
pages/
  index.jsx          — main app (all four tabs)
  login.jsx          — login / signup
  api/
    configure.js     — calls Anthropic for battery configurations
    specsheet.js     — calls Anthropic for spec sheet content
    library.js       — GET/PUT shared component library
    history.js       — GET/POST configuration history
    settings.js      — GET/PUT per-user settings

lib/
  guardrails.js      — banned phrase validation, system prompts
  demoLibrary.js     — default library used before catalog import
  supabase.js        — browser + server Supabase clients

supabase/
  schema.sql         — run this once in Supabase SQL Editor

styles/
  globals.css        — all styles

middleware.js        — redirects unauthenticated users to /login
```

---

## Adding Users

Users create their own accounts at the login page. There's no admin invite required — just share the URL. If you want to lock it down to specific email domains, add a check in `middleware.js`.
